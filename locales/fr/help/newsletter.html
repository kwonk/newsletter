Ce plugin permet de gérer la rédaction et l'envoi de newsletters ainsiq que la liste des abonnés.

Lettres

Actions sur les lettres

Il est possible d'effectuer une série d'actions simultanément sur plusieurs lettres.

Newsletter

  • envoyer : envoie la newsletter

Email à envoyer

  • Newsletter : envoie la newsletter
  • Activation : envoie le mail pour activer le compte
  • Confirmation : envoie la confirmation de création du compte
  • Désactivation : envoie le mail de suppression du compte

Abonnés

Actions sur les abonnés

Il est possible d'effectuer une série d'actions simultanément sur plusieurs abonnés.

Email à envoyer :
  • Newsletter : envoie la newsletter
  • Activation : envoie le mail pour activer le compte
  • Confirmation : envoie la confirmation de création du compte
  • Désactivation : envoie le mail de suppression du compte
Changer l'état :
  • Activer : active les comptes
  • Désactiver : désactive les comptes
  • Supprimer : supprime les comptes
Changement du format :
  • html : change le format d'envoi en html pour les comptes sélectionnés
  • texte : change le format d'envoi en texte pour les comptes sélectionnés
Raz dernier envoi :
  • Dernier envoi : réinitialise la date du dernier envoi à la date d'inscription de l'abonné
Ajouter un abonné
  • Email : ajoute l'email défini à la liste des abonnés

Propriétés

Sont ici regroupés tous les paramètres d'administration des newsletters.

Résumé
Statistiques des abonnés :
Affiche l'état des abonnés.
Statistiques des billets de type newsletter :
Affiche l'état des newsletters.
Paramètres
Paramètres avancés :
  • Nom de l'éditeur : défini le nom de la personne qui emet la newsletter
  • Email de l'éditeur : défini le courriel de la personne qui emet la newsletter
  • Code visuel : ajout d'un code visuel de vérification pour éviter les robots
  • Auto-activation des abonnés : cocher cette case pour valider automatiquement les souscriptions
  • Utiliser le format par défaut pour l'envoi : défini si on utilise le format d'envoi par défaut ou par utilisateur
  • Format par défaut pour l'envoi : défini le format d'envoi par défaut
  • Accusé de réception : Cocher cette case pour recevoir un accusé de réception à chaque envoi de mail pour chaque destinataire
  • Restreindre l'aperçu à l'extrait des billets : afficher uniquement les extraits des billets de type newsletter
  • Affiche le contenu des billets : affiche le contenu des billets dans la newsletter
  • Taille maximale du contenu des billets : définir la taille du contenu des billets dans la newsletter
  • Affiche le contenu au format texte : cocher cette case pour que les les billets de la newsletters soient convertis en texte
  • Affiche les miniatures : cocher cette case pour afficher une image pour chaque billet de la newsletter
  • Taille des miniatures : définir la taille des images
  • Utiliser l'option suspendre : cocher cette case pour utiliser l'option de suspension de compte
  • Choix de la date pour le tri des billets : permet de sélectionnés le type de date utilisé pour filtrer la sélection des billets
  • Créer automatiquement un abonné lorsqu'un utilisateur est ajouté dans le plugin Agora : lien avec le plugin agora
  • Format de la date dans l'info des billets : définir le format de la date des billets de la newsletter (exemple= %A %d %B %Y - %H:%M)
Paramètres pour la lettre automatique contenant la liste des newsletters
  • Date du précédent envoi : affiche les date et heure du dernier envoi automatique/li>
  • Envoi automatique lors de la création d'un billet : cocher cette case pour activer l'envoi de la newsletter suite à la création d'un billet
  • Envoi automatique lors de la modification d'un billet : cocher cette case pour activer l'envoi de la newsletter suite à la publication ou la modification d'un billet
  • Nombre minimum de billets : définir le nombre minimum de billets à afficher dans la newsletter
  • Nombre maximal de billets : définir le nombre maximum de billets à afficher dans la newsletter
    • Note de jcp : J'ai trouvé pourquoi les newsletter ne partaient pas.. c'est à cause des champs (Nombre minimum de billets, Nombre maximal de billets). Cela veut dire, nombre minimum de billet "AVANT ENVOI" !! Effectivement si on monte le minimum à 5, la newsletter attend qu'il y ait 5 billets en attente avant l'envoi .. j'ai mis la valeur à 1 et cela part de suite !
  • Catégorie : filtre les billets à envoyer dans la catégorie sélectionnée
  • Inclure les sous-catégories : cocher cette case permet d'inclure les billets des sous-catégories de la catégorie définie ci-dessus
  • Ajoute la date dans le titre de la lettre d'information : cocher cette case pour ajouter la date dans le titre de la newsletter à la suite du "Sujet de la Newsletter"
Paramètres de la page publique contenant la liste des newsletters
  • Nombre par page : Nombre de newsletters affichées sur la page
  • Trier par : Choix du champ de tri des billets à afficher
  • Ordre : Choix Cocher cette case pour activer l'envoi de la newsletter suite à la publication d'un billet
Planification

Cet onglet permet de définir une tâche planifiée pour l'envoi de la newsletter en utilisant le plugin nommée dcCron pour Dotclear 2.

Planification de la newsletter :
  • Intervalle de temps en secondes entre 2 exécutions : défini le temps entre chaque envoi
  • Date de la première exécution : spécifie la date du premier envoi
  • Planifier : crée la tÃ¢che planifiée NewsletterPlan avec les paramètres précédents
  • Déplanifier : supprime la tÃ¢che planifiée NewsletterPlan
Tâche planifiée : NewsletterPlan
Affiche les caractéristiques de la tâche planifiée.
  • Etat : affiche l'état actuel de la tâche planifiée
  • Intervalle : affiche l'intervalle actuellement défini
  • Dernière exécution : affiche la date de la dernière exécution
  • Prochaine exécution : affiche la date de la prochaine exécution
  • Temps restant : affiche le temps restant avant la prochaine exécution
  • Activer : active la tâche planifiée NewsletterPlan
  • Désactiver : désactive la tâche planifiée NewsletterPlan

Maintenance
Etat de l'extension :
Cocher cette case pour activer l'extension.
Export des données :
Choisir le mode pour l'export de la liste des abonnés.
  • Ce blog seulement : crée le fichier public/nomblog-newsletter.dat
  • Toutes les données : crée le fichier public/newsletter.dat
Import des données :
Choisir le mode pour l'import de la liste des abonnés.
  • Ce blog seulement : importe le fichier public/nomblog-newsletter.dat
  • Toutes les données : importe le fichier public/newsletter.dat
Importe une liste d'abonnés à partir d'un fichier texte dans le blog courant :
  • Importer : lance l'import du fichier sélectionné
Adapter le template pour le thème :
Créer une adaptation du template générique pour s'adapter correctement au thème.
  • Nom du thème : sélectionner le thème désiré (ne sont disponibles que les thèmes avec une adapation du home.html et du post.html)
  • Adapter : lancer la création du fichier template subscribe.newsletter.html dans le tpl du thème
  • Cliquer ici pour tester le template : test de l'affichage du template subscribe.newsletter.html
Suppression de toutes les informations concernant newsletter dans votre base de données :
Suppression : cliquer sur ce bouton pour désactiver l'extension, supprimer la table (prefix)_newsletter, supprimer les valeurs du paramétrage dans settings et supprimer la version.

Configuration des widgets

Mise en place du widget Newsletter :
Ce widget affiche le formulaire d'abonnement à la newsletter
Déplacer le widget "Newsletter" vers l'un des bandeaux.
Paramétrage du wigdet :
  • Titre : définition du titre
  • Afficher le titre : cocher cette case pour afficher le titre du widget
  • Dans un widget : cocher cette case pour afficher le formulaire d'abonnement directement sur la page d'accueil
  • Dans un lien : cocher cette case pour afficher un lien vers la page du formulaire d'abonnement
  • Titre du lien de souscription : définition du lien vers le formulaire d'abonnement
  • Page d'accueil uniquement : cocher cette case pour afficher sur la page d'accueil uniquement
Mise en place du widget Newsletters :
ce widget affiche la liste des newsletters envoyées et à l'état publié
Déplacer le widget "Newsletters" vers l'un des bandeaux.
Paramétrage du wigdet :
  • Titre : définition du titre
  • Afficher le titre : cocher cette case pour afficher le titre du widget
  • Limite : nombre de newsletter par page
  • Order by : champ utilisé pour le tri des newsletters
  • Trier : ordre de tri
  • Page d'accueil uniquement : cocher cette case pour afficher sur la page d'accueil uniquement
